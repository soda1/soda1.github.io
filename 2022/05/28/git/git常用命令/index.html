文件的四种状态

版本控制就是对文件的版本控制，要对文件进行修改、提交等操作，首先要知道文件当前在什么状态，不然可能会提交了现在还不想提交的文件，或者要提交的文件没提交上。

  • Untracked: 未跟踪, 此文件在文件夹中, 但并没有加入到git库, 不参与版本控制. 通过git add 状态变为Staged.
  • Unmodify: 文件已经入库, 未修改, 即版本库中的文件快照内容与文件夹中完全一致. 这种类型的文件有两种去处, 如果它被修改, 而变为Modified. 如果使用git rm移出版本库, 则成为Untracked文件
  • Modified: 文件已修改, 仅仅是修改, 并没有进行其他的操作. 这个文件也有两个去处, 通过git add可进入暂存staged状态, 使用git checkout 则丢弃修改过, 返回到unmodify状态, 这个git checkout -- file 即从库中取出文件, 覆盖当前修改 !
  • Staged: 暂存状态. 执行git commit则将修改同步到库中, 这时库中的文件和本地文件又变为一致, 文件为Unmodify状态. 执行git reset HEAD filename取消暂存, 文件状态为Modified
查看帮助文档
git [command] --help
初始化本地仓库
git init

git克隆远程仓库
git clone https/ssh

git绑定远程仓库
git remote add origin ssl/hhtp
#设置上传分支并上传
git push --set-upstream origin master
提交三板斧
#查看指定文件状态
git status [file-name]
#查看所有文件状态
git status
# 添加所有文件到暂存区
git add .
# 提交暂存区中的内容到本地仓库 -m 提交信息
git commit -m "消息内容"   
#先拉取远程仓库的最新内容,防止发生冲突
git pull
#同步到远程仓库
git push

查看历史版本
#查看单个文件的历史版本
git log <file-name>
#查看提交的历史版本
git log
checkout使用
#从库中取出文件覆盖修改
git checkout -- file-name  
#将文件回退到某个版本
git checkout commit_id -- file-name

取消暂存
git reset HEAD file-name
#取消所有暂存
git restore --staged .

取消更改
#丢弃工作区更改内容
git restore file
diff差异比较
#不加参数即默认比较工作区与暂存区
git diff
#比较暂存区与最新本地版本库（本地库中最近一次commit的内容）
git diff --cached [<path>...]
#比较工作区与最新本地版本库。如果HEAD指向的是master分支，那么HEAD还可以换成master
git diff HEAD [<path>...]
#比较工作区与指定commit-id的差异　　　　　　
git diff commit-id [<path>...]
#比较暂存区与指定commit-id的差异
git diff --cached [<commit-id>] [<path>...]
#比较两个commit-id之间的差异
git diff [<commit-id>] [<commit-id>]
发生错误了想回退某一版本

  • reset 

    # 先查看提交历史，确认需要回退的版本
git log
# 进行版本回退，回到指定版本的状态
# --hard:
# Resets the index and working tree. Any changes to tracked files in the working tree since <commit> are discarded
git reset --hard commit_id
# 强推送至远程分支
git push  --force

    回到指定某一个版本，但是此版本之后的提交都会消失，好像没有发生过一样。幸运的是 git reflog 会记录下所有的commit_id，因此还是可以在任意版本中穿梭

    # 查看操作的所有记录
git reflog
# 回退到指定版本
git reset commit_id
#重新提交三板斧
git add.
git commit -m ' '

  • revert

    对已经commit的代码再commit一次，对已提交的代码没有影响

    #先查看提交历史，确认需要回退的版本
git log
#进行版本回退，回到指定版本的状态
git revert commit_id
#如果有争议想丢掉这次revert，可以使用
git revert --abort

    revert可能会产生争议参考https://stackoverflow.com/questions/13008170/im-not-able-to-revert-a-file-while-its-conflicting-git

修改最后一次本地提交的消息

如果最后一次commit已经push了那么就应当不要再做修改了（修改后再次push会报错的）。

git commit -amend
git 分支常用命令
# 列出所有本地分支
git branch
# 列出所有远程分支
git branch -r
# 新建一个分支，但依然停留在当前分支
git branch [branch-name]
# 新建一个分支，并切换到该分支
git checkout -b [branch]
#切换分支
git checkout branch-name
# 合并指定分支到当前分支
$ git merge [branch]
# 删除分支
$ git branch -d [branch-name]
# 删除远程分支
$ git push origin --delete [branch-name]$ git branch -dr [remote/branch]
